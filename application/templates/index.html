Accessing GOV services

Static prototypes for the various pages and flow through the proposed broker.

Pages

Broker pages

  • Confirm IDP routing

    We shouldn't automatically route the user to an IDP without getting them to confirm because we have no control if we send them to the wrong one. They might get stuck.

  • Choose from a reduced list of IDPs

    Similar to above but present a choice of IDPs the user could belong to.

  • Check if user knows their email

    We can use the email to filter the list of IDPs so see if they user knows theirs.

  • IDP choice based on email

    Given the email provided above give users a choice of IDP/department.

  • IDP choice based on email being similar to another

    Users might make mistakes entering their email, try to mitigate against these. If email looks similar to other departments then suggest those.

  • User changes email address

    Need to give user ability to re-enter email address incase they put an incorrect one in.

  • Find and select from complete list of IDPs

    The route if we have no information to use to filter the list of IDPs automatically. Present full list to user.

  • Ask user if first time using broker

    Optional: check to see if user has used the broker before. Similar approach to Verify. Depends on how visible we want the broker to be. Does the user need to know that much about it?
